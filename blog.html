var - let - const

Actually we use many variables. Especially we use a lot to do math. In programming we use variables a lot. Three variables have been used so far in JSS of time and need. Var, Let, Const

Now try to know separately

var

  • "var" is like an independent variables
  • If declared within a block - like "an if statement", they are still accessible outside that block.
  • They can be redeclare within the same scope without error.
  • "var" is globally scoped, but not block-scoped.

let

  • "let" is a block-scoped variable
  • They are not hoisted to the top of the block, so accessing them before their declaration results in a ReferenceError.
  • They can be reassigned after declaration within the same scope.
  • They cannot be redeclare within the same block.

var

  • "const" is a block-scoped variable
  • They must be assigned a value at the time of declaration
  • They cannot be reassigned after declaration, making them suitable for values that should not change.
  • Like let, they are not hoisted and accessing them before declaration results in a ReferenceError.

Undefined and Null Use Cases

Undefined

A variable that is declared but not assigned a value automatically holds the value undefined.When an object property doesn't exist or a function parameter is missing, its value is undefined.

  • As a default value for function parameters..
  • To check if a variable has been assigned a value.
  • When an expected value is not available, such as when fetching data from APIs.
  • To indicate the absence of a property in an object.

Null

Null is a value that represents the intentional absence of any object value. Null is explicitly assigned to indicate that a variable has no value

  • To explicitly set a variable or object property to signify the absence of a value.
  • As a way to reset or clear an object property.

REST APIs

REST API stands for Representational State Transfer Application Programming Interface. It is a set of architectural principles and constraints used to design and develop web services that allow different software applications to communicate with each other over the internet.

REST is not a protocol, but rather a style of designing networked applications. It's built upon a few key principles:

Client-Server Architecture, Stateless, Uniform Interface, Code on Demand

REST APIs are commonly used for building web services that provide data and services to other applications. They use HTTP methods to perform actions like retrieving data, creating new resources, updating existing resources, and deleting resources. Data is usually exchanged in common formats such as JSON or XML.
